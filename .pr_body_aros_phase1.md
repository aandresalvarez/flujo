AROS v2 — Phase 1: Wrapper catch+repair, built-ins, linter --fix

Summary

- Catch provider JSON-mode failures at the wrapper (pydantic_ai.exceptions.UnexpectedModelBehavior) and route them through the existing deterministic→LLM repair path.
- Add two minimal built-ins to simplify context merging and demos: flujo.builtins.wrap_dict and flujo.builtins.ensure_object.
- Add fixer for V-A8 so `flujo validate --fix` can enable `processing.structured_output: openai_json` when a schema is present. Includes dry‑run patch preview.
- Update FSD.md (FSD-AROS-001) with Phase 1 status.

Key Changes

- flujo/agents/wrapper.py
  - Treat UnexpectedModelBehavior like ValidationError/ModelRetry for auto-repair.
- flujo/builtins.py
  - New skills: `wrap_dict(key='value')`, `ensure_object(key='value')`.
- flujo/validation/fixers.py
  - New fixer: V-A8 (enable JSON mode when schema present); integrated with --fix and --fix-dry-run.
- Tests
  - tests/unit/aros/test_wrapper_unexpected_behavior.py
  - tests/unit/builtins/test_builtins_adapters.py
  - tests/unit/cli/test_fix_va8.py
- Docs
  - FSD.md: Phase 1 status section.

Motivation

Provider JSON-mode can fail early, bypassing current repair. This increases flakiness in structured steps and friction in demos. Phase 1 addresses the highest-impact gap while remaining fully backwards compatible.

Validation

- New unit tests pass locally for the added functionality.
- No changes to public APIs; fixes are behind existing flags/paths.

Follow-ups (Phase 2/3 per FSD)

- Streaming aggregation and schema-aware coercion.
- Adaptive fallback (retry → extractor → degrade).
- Optional schema simplifier.

Notes

- The V-A8 fixer uses safe YAML parsing and may reformat the step block; changes are previewable via `--fix-dry-run` before applying.
